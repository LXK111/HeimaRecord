## 1. 文档定位

本文用于指导 `heima-record` 当前赛事编排能力的内测验收，并规划下一阶段“赛事结果导出增强”。

当前已支持的赛制：

- 小组循环 + 单败淘汰赛
- 瑞士轮 + 单败淘汰赛
- 直接单败淘汰赛
- 双败淘汰赛

当前暂缓的赛制：

- 三败淘汰赛

暂缓原因：三败的核心价值低于当前现场验收和结果导出增强，继续扩展赛制会增加测试面，但不能直接提升近期交付质量。

## 2. 验收原则

验收不只看“能生成场次”，还要看现场是否能安全完成一次赛事闭环。

每个赛制至少验证：

- 选手导入或录入是否正确。
- 编排生成是否符合赛制。
- 场次能否进入比赛控制台。
- 计时、计分、警告、申诉、赛后修正是否可用。
- 刷新浏览器后数据是否保留。
- 推进下一轮时是否只使用已完成结果。
- 结果导出是否能满足归档和复核。

## 3. 通用验收清单

### 3.1 基础数据

- 导入 Excel 选手名单。
- 导入 CSV 选手名单。
- 粘贴录入选手名单。
- 检查姓名、单位、种子序号是否正确。
- 切换赛制前后，旧分组、旧签表、旧轮次不会误导当前赛制。

### 3.2 比赛控制台

- 未开始、进行中、暂停、结束状态正确。
- 计时开始、暂停、重置可用。
- 刷新浏览器后当前场次和时间状态不丢失。
- 直接计分、部位计分、限制回合计分按规则工作。
- 双方同时得分和双方同时判罚可用。
- 违规警告、黄牌、红牌等记录正确。
- 判罚中止能区分红胜、蓝胜、平局。
- 申诉时间点能被记录。
- 撤销操作不回退计时。
- 赛后分值和胜方只能通过赛后修正入口修改。

### 3.3 数据持久化

- 刷新浏览器后选手、规则、场次、比分、签表仍存在。
- 关闭浏览器后重新打开，最近状态可恢复。
- 导出备份 JSON 后可重新导入恢复。
- 清理浏览器数据后，仅能依赖备份 JSON 恢复。

## 4. 分赛制验收清单

### 4.1 小组循环 + 单败淘汰

建议用例：

- 12 人，配置每组 6 人，每组出线 2 人，总晋级 4 人。
- 13 人，验证分组人数不均时的展示。

验收项：

- 按种子蛇形分组。
- 每组生成完整单循环。
- 小组排名按配置规则排序。
- 胜场优先、积分优先、净胜分、相互胜负开关生效。
- 晋级线同分时能标记附加赛。
- 小组赛完成后能生成淘汰签表。
- 半决赛后按配置生成季军赛。

### 4.2 瑞士轮 + 单败淘汰

建议用例：

- 20 人，配置 5 轮，晋级 12 人。
- 21 人，验证奇数人数轮空。

验收项：

- 第 1 轮按种子生成。
- 当前轮全部完成前不能生成下一轮。
- 锁定当前轮后才能生成下一轮。
- 后续轮次按当前排名生成。
- 不出现重复对阵。
- 奇数人数轮空只给赛事积分，不计真实胜场。
- 瑞士轮完成后能生成淘汰签表。

### 4.3 直接单败淘汰赛

建议用例：

- 8 人，无轮空。
- 10 人，验证高种子轮空。

验收项：

- 不需要预赛排名。
- 所有正常参赛选手直接进入签表。
- 按种子生成首轮。
- 非 2 的幂人数自动给高种子轮空。
- 推进淘汰赛直到冠军产生。
- 季军赛开关生效。

### 4.4 双败淘汰赛

建议用例：

- 8 人标准双败。
- 10 人验证胜者组首轮轮空。

验收项：

- 首轮进入胜者组。
- 输 1 场后进入败者组。
- 输 2 场后淘汰。
- 当前待赛全部完成前，不生成下一波场次。
- 败者组奇数人数时能轮空。
- 胜者组冠军和败者组冠军进入总决赛。
- 总决赛一场决胜，不生成决赛重置。
- 总决赛结束后赛事完成。

## 5. 结果导出现状

当前结果导出更偏“场次结果导出”，适合查看每场比赛的：

- 场次编号
- 对阵双方
- 比分
- 警告和处罚
- 胜方
- 结束原因

当前不足：

- 缺少赛事维度最终名次。
- 缺少冠军、亚军、季军汇总。
- 缺少小组排名导出。
- 缺少瑞士轮最终排名导出。
- 缺少淘汰签表节点导出。
- 双败只展示签表推进，不导出负场累计和淘汰路径。

## 6. 结果导出增强方案

### 6.1 增强目标

新增“赛事结果导出”，与现有“场次结果导出”并存。

二者区别：

- 场次结果导出：记录每场比赛发生了什么。
- 赛事结果导出：说明本次赛事最终排名和晋级过程。

### 6.2 导出内容

建议 Excel 文件包含以下工作表：

- `赛事摘要`
- `最终名次`
- `预赛排名`
- `淘汰签表`
- `全部场次`
- `警告处罚`

CSV 可以保持单表导出，优先导出 `最终名次` 或 `全部场次`，避免 CSV 多表结构不清晰。

### 6.3 赛事摘要

字段建议：

| 字段 | 含义 |
|------|------|
| 赛事名称 | 当前赛事名称 |
| 赛制 | 当前选择的赛制 |
| 参赛人数 | 正常参赛选手数量 |
| 场次数 | 已生成场次数 |
| 已完成场次数 | 已结束场次数 |
| 冠军 | 最终冠军 |
| 亚军 | 最终亚军 |
| 季军 | 有季军赛时填写 |
| 导出时间 | 本地导出时间 |

### 6.4 最终名次

字段建议：

| 字段 | 含义 |
|------|------|
| 名次 | 最终名次 |
| 选手 | 选手姓名 |
| 单位 | 俱乐部或单位 |
| 阶段结果 | 冠军、亚军、季军、八强等 |
| 负场数 | 双败赛制使用 |
| 备注 | 退赛、弃权、判罚中止等 |

第一版名次策略：

- 小组 + 淘汰、瑞士 + 淘汰、直接单败：以淘汰签表最终节点推导冠军、亚军、季军。
- 双败：以总决赛胜者为冠军，负者为亚军，其余选手先按被淘汰轮次和负场数粗排。
- 没有完整结束的赛事导出时，标记为“未完赛结果”。

### 6.5 预赛排名

适用赛制：

- 小组循环 + 单败淘汰赛
- 瑞士轮 + 单败淘汰赛

字段建议：

| 字段 | 含义 |
|------|------|
| 分组或阶段 | 小组名或瑞士轮 |
| 名次 | 预赛名次 |
| 选手 | 选手姓名 |
| 积分 | 赛事积分 |
| 真实胜场 | 不含轮空 |
| 平局 | 平局数 |
| 负场 | 负场数 |
| 净胜分 | 得分减失分 |
| 纪律扣分 | 警告、黄牌、红牌折算 |
| 是否晋级 | 是或否 |
| 是否需要附加赛 | 是或否 |

### 6.6 淘汰签表

字段建议：

| 字段 | 含义 |
|------|------|
| 阶段 | 单败、季军赛、胜者组、败者组、总决赛 |
| 轮次 | 第几轮 |
| 节点 | 签表节点名称 |
| 红方 | 红方选手 |
| 蓝方 | 蓝方选手 |
| 比分 | 红蓝比分 |
| 胜方 | 胜方 |
| 状态 | 待比赛、已完成、轮空 |

## 7. 分步推进建议

### 阶段 E1：赛事结果 Excel 导出

实施状态：已完成。

目标：

- 增加一个“导出赛事结果 Excel”按钮。
- 输出 `赛事摘要`、`最终名次`、`预赛排名`、`淘汰签表`、`全部场次`。

验收标准：

- 小组 + 淘汰赛能导出冠军、亚军、季军和小组排名。
- 瑞士 + 淘汰赛能导出瑞士轮排名和淘汰结果。
- 直接单败能导出签表结果。
- 双败能导出总决赛结果和负场数。

已落地边界：

- 赛事未完成时明确标记“未完赛结果”，不把当前领先者误写为冠军。
- 单败类赛制第一版稳定输出冠军、亚军和季军，其余完整名次留待后续完善。
- 双败第一版输出总决赛冠亚军与各选手负场数，完整 1 至 N 名次仍由阶段 E2 负责。
- 直接单败和双败没有预赛阶段时，保留空的 `预赛排名` 工作表及表头，保证工作簿结构一致。

## 8. 推荐下一步

当前 E1 和 E3 已完成。下一步先依据随包清单完成现场内测，再根据实际归档要求决定是否推进 E2。

原因：

- E2 需要先确定同轮淘汰选手的并列或继续排序规则，避免生成没有规则依据的完整名次。
- 继续增加三败会扩大测试面，仍不进入当前路线。
- 现场内测结果应优先于新增赛制。
