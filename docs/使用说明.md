## 打开方式

进入 `heima-record` 文件夹，双击 `index.html`。该文件已经内置页面样式和程序代码，不需要 Node.js。

推荐浏览器：

- Chrome
- Edge
- Safari

## 导入场次

进入“场次”，点击“导入场次”选择 Excel 或 CSV 文件；电脑端也可以把文件直接拖到场次页面任意内容区域。页面出现“松开导入比赛场次”遮罩后松开文件即可导入。导入会替换当前全部场次，并清空已有赛事编排；已有数据时系统会要求二次确认。

导入文件首行建议包含：

- 场次编号
- 组别
- 场地
- 红方姓名
- 红方单位
- 蓝方姓名
- 蓝方单位

也可以参考同目录下的 `示例导入模板.csv`。

普通导入场次支持按场移除，也支持点击分组标题右侧的删除按钮移除整个分组。赛事编排生成的场次不能在此单独删除，需要到“编排”统一清空，避免签表失去完整性。

## 赛事首页

进入“首页”可以查看当前赛制、正常参赛人数、赛事阶段、已完成场次和总进度。

“重置赛事”会清空选手、场次、比分、排名和签表，并恢复默认赛制配置，但保留当前计分规则。该操作不可撤销，执行前应先导出完整备份 JSON。

首页显示本次赛事唯一的“赛事 ID”。完成编排后可点击“开始赛事”；开始后计分规则、排名规则、纪律扣分、赛制参数和阶段规则全部冻结。即使没有点击该按钮，真实开始第一场赛事比赛时也会自动开始赛事并冻结规则。

## 赛事编排

进入“选手”录入或导入所有选手，再进入“赛事”选择赛制并生成比赛。电脑端可以把名单文件拖到选手页面任意内容区域，手机端点击“导入选手”后通过系统文件选择器选择文件。

发布包中的 `16人选手测试数据.xlsx` 和 `32人选手测试数据.xlsx` 可以直接在“选手”页面导入。两份文件均使用首个工作表“选手名单”，包含姓名、单位和连续种子序号，适合快速验证不同规模的编排结果。

当前支持：

- 小组循环 + 单败淘汰赛
- 瑞士轮 + 单败淘汰赛
- 直接单败淘汰赛
- 双败淘汰赛

小组和瑞士轮赛制支持配置晋级人数；淘汰赛支持季军赛。双败采用一场总决赛直接决出冠军，不生成决赛重置。

小组循环可选择“按每组人数”或“按小组数量”分组。“每组保底出线”先保证每组的固定名额，“淘汰赛目标人数”再按全部未保底选手的排名补足；目标人数不能小于“实际小组数 x 每组保底出线人数”。多人在晋级边界完全同分时仍使用附加赛。

阶段比赛规则分为三套：小组赛、瑞士轮和附加赛使用“预赛规则”；普通淘汰轮使用“淘汰赛规则”；决赛和季军赛使用“冠亚季殿赛规则”。每套规则可独立配置比赛时长和目标分。

“场地数量”适用于全部赛制，支持 1 至 26 个场地。生成编排时系统自动分配场地：小组循环的同组全部比赛固定在同一场地，不同小组按组内场次数尽量均衡；瑞士轮、附加赛和淘汰赛按当前累计场次数分配到负载较少的场地。场地少于小组数时，一个场地可以承载多个完整小组。

“采用种子编排”默认关闭，并对小组循环、瑞士轮首轮、直接单败和双败首轮统一生效：

- 关闭时，生成动作会随机打乱选手并固定本次分组或签位。
- 开启时，使用名单中的种子序号进行蛇形分组或上下半区配对。
- 随机只在生成编排时执行一次，结果会随赛事状态保存；刷新页面不会重新随机，清空后重新生成才会重新抽签。
- 奇数人数瑞士轮仍按抽签后的顺序选择本轮轮空选手，并避免同一选手重复轮空。
- 第 2 轮开始仍按全局排名、积分接近、同单位回避和禁止重复对阵规则编排。

小组循环可勾选“小组赛尽量避开同单位”。系统首先保证各组人数均衡，再尽量减少同单位选手进入同组；当单位人数分布使完全规避不可能时，会保留无法规避的冲突并在生成后提示。

瑞士轮仍按全部选手统一配对、统一排名和统一出线，仅按实际场地生成“瑞士A组”“瑞士B组”等现场分组，选手下一轮可能进入不同现场组。

生成比赛后，“选手分组结果”会输出：

- 小组循环：按 A组、B组等展示固定选手名单、单位和组内序号。
- 瑞士轮：按每一轮展示瑞士A组、瑞士B组等现场分组，以及场次编号、对阵、场地和轮空选手。

点击“导出分组结果”可生成 Excel。小组循环导出一张完整名单表；瑞士轮按轮次生成独立工作表，适合打印或发送给选手核对。

“编排结果”可以阅览全部赛事生成场次。未开始场次允许修改场次编号、现场分组和场地，不允许修改红蓝双方；已开始或已结束场次只读。修改现场分组或场地后，选手分组结果和导出内容会同步更新。“导出编排 Excel”输出“编排场次”“赛事信息”和“场地统计”三个工作表，包含赛事 ID、场次 ID 和规则信息，可供不同场地的记录台直接导入。“清空编排”会删除赛事生成场次及其比赛结果，但保留选手、规则、赛制配置和普通导入场次。

外部验收时请同时查看同目录下的 `赛事验收清单.pdf`。

## 比赛记录

进入“场次”，选择比赛后进入“控制台”。

控制台支持：

- 开始、暂停、重置计时
- 按击中部位自动计分
- 限制回合模式下按回合记录红方得分、蓝方得分、双方得分或无效回合
- 使用“综合判定”一次提交双方得分和双方各类警告次数
- 按警告分级记录违规
- 记录申诉时间点
- 撤销上一步操作，撤销不会回退当前计时
- 重置当前比赛
- 手动判定红方胜、蓝方胜或平局
- 自动记录操作日志
- 当前比赛结束后，点击“下一场”进入同一场地编号最靠前的待赛比赛；比赛未结束或同场地没有待赛比赛时按钮不可用

比赛结束后，普通计分和警告按钮会锁定。如需处理申诉或仲裁，请使用“赛后修正”，填写原因后再修改比分或胜方。

## 规则配置

进入“规则”，可以点击选择文件，也可以把规则文件拖到规则页面任意内容区域；当前规则仍可导出。

推荐使用同目录下的 `规则配置模板.xlsx`，包含 4 个工作表：

- `基础规则`
- `部位分值`
- `警告分级`
- `警告转换`

也可以导入单个 CSV 规则表。CSV 会按表头自动识别类型。

`基础规则` 中的 `计分模式` 支持：

- `target_score`：目标分模式，保留原有部位计分能力
- `round_limit`：限制回合模式，按最大回合数记录每个交剑回合，时间到或回合数打满后按比分判胜

`警告分级` 可以配置：

- `扣分`：本级警告触发时扣除被警告方分值
- `是否处罚`：是否累计到处罚次数
- `是否中止比赛`：是否当次直接中止比赛
- `中止结果`：支持 `对方胜`、`本人胜`、`平局`、`手动判定`

当时间结束或限制回合打满且比分相同，如启用加时，系统会进入加时并暂停等待裁判开始计时。加时仍可继续记录综合判定和回合结果。

## 本地保存

系统会自动保存到当前浏览器的本地数据中。

刷新页面、关闭后重新打开，通常不会丢失数据。

注意：

- 清理浏览器站点数据会删除本地记录
- 更换浏览器或更换电脑不会自动带走数据
- 比赛现场建议定期导出完整备份 JSON

## 完整备份和恢复

进入“结果”，使用“完整备份”区域。备份 JSON 可以点击选择，也可以直接拖到结果页面任意内容区域：

- “导出备份 JSON”：保存规则、场次、比分、计时状态和操作记录
- “导入备份 JSON”：恢复之前保存的完整状态

完整备份用于恢复现场状态；成绩 Excel/CSV 只用于提交结果。

手机端应使用 Safari、Chrome 或 Edge 打开页面。部分文件管理器预览、微信内置浏览器或其他内置 WebView 不提供系统文件选择能力，此时页面可以显示，但点击导入不会弹出文件选择器，需要改用系统浏览器打开。

## 结果导出

进入“结果”，可以导出：

- CSV 成绩表
- 场次 Excel 成绩表
- 赛事结果 Excel：包含赛事摘要、最终名次、预赛排名、淘汰签表和全部场次

分场地比赛完成后，在各场地记录台导出 Excel 或 CSV 结果，再在总控“结果”页点击“导入比赛结果”。系统只合并赛事 ID 和场次 ID 都一致的完赛结果：完全相同的重复结果会忽略；总控已有不同完赛结果时只提示冲突并保留总控结果，不会自动覆盖。

比赛结束后必须导出结果文件作为正式留档。

未完成赛事导出时会明确标记“未完赛结果”。当前完整 1 至 N 名次仍属于 E2 TODO，现阶段稳定输出冠亚季军和双败负场数。

## 无 Node 说明

这个交付包运行时不需要 Node.js、npm、数据库或网络。

开发和重新打包仍需要开发环境；比赛现场只需要浏览器。
