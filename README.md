## 项目定位

`heima-record` 是一个本地 Web 端兵击赛事控制台，覆盖选手导入、赛事编排、现场计时记分、规则判定、本地保存和结果导出。

## 当前能力

- 导入 Excel / CSV 场次表
- 赛事首页展示赛制、参赛人数、阶段和完成进度，并支持保留规则的一键重置
- 配置比赛时长、目标分、平局、加时和处罚扣分规则
- 单场比赛控制台：计时、加减分、处罚、手动结束
- 浏览器 IndexedDB 本地保存
- 导出 CSV / Excel 比赛结果
- 导出 / 导入完整 JSON 备份
- 通过 CSV / XLSX 配置部位分值、警告分级和警告转换
- 支持申诉记录、撤销、重置比赛和赛后修正
- 支持目标分模式和限制回合模式
- 支持综合判定、判罚中止比赛结果配置和加时
- 支持小组循环 + 单败、瑞士轮 + 单败、直接单败和双败淘汰赛
- 支持阅览、编辑、清空和导出编排结果；系统按场地数量自动均衡分配赛事场次
- 支持展示并导出小组循环固定名单和瑞士轮逐轮现场分组结果
- 默认忽略种子随机抽签，可统一启用种子编排；随机结果生成后持久化
- 小组赛可尽量规避同单位，且同组全部场次固定在同一场地
- 支持在场次页导入、按场删除或按组删除普通场次
- 支持把文件拖到对应功能页面导入选手、场次、规则和完整备份，也可点击按钮并重复选择同一文件
- 支持排名规则优先级、纪律扣分、附加赛和季军赛
- 导出赛事摘要、最终名次、预赛排名、淘汰签表和全部场次
- 小组赛可按每组人数或小组数量分组，并采用“每组保底出线 + 淘汰赛目标人数”语义
- 预赛、淘汰赛和冠亚季殿赛可分别配置比赛时长与目标分
- 赛事开始后冻结计分、排名和阶段规则；控制台结束当前比赛后可进入同场地下一场
- 综合判定可同时录入双方得分和每一种警告，警告转换会消费来源警告
- 编排和结果文件携带稳定赛事 ID、场次 ID；结果导入只合并同赛事数据，冲突只提示不覆盖

## 导入字段

导入文件首行建议包含以下字段，中文或英文表头均可：

- 场次编号 / matchNo
- 组别 / group
- 场地 / piste
- 红方姓名 / redName
- 红方单位 / redClub
- 蓝方姓名 / blueName
- 蓝方单位 / blueClub

## 本地运行

```bash
npm install
npm run dev
```

赛事领域验收测试：

```bash
npm run test
```

## 生成无 Node 交付包

```bash
npm run release
```

生成目录：

```text
release/heima-record/
```

比赛现场使用者只需要浏览器，双击发布包里的单文件 `index.html` 即可打开。

发布包固定包含 7 个文件：单文件程序、PDF 使用说明、PDF 赛事验收清单、场次示例、规则配置模板，以及 16 人和 32 人选手测试工作簿。`npm run release` 会在打包后自动检查文件数量、PDF 格式和测试名单结构。

源说明修改后先执行 `npm run release:docs` 更新 PDF，再执行 `npm run release` 生成交付包。PDF 生成仅用于开发维护，需要 Python 3、ReportLab 和可嵌入的中文字体；比赛现场仍然不需要 Node.js 或 Python。
